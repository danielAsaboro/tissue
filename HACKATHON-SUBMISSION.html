Copy-paste project description

Sports traders face a hard problem in-play: a score event changes fair value immediately, but the consensus market reacts over seconds, data can go stale, and a strategy that looks good in a backtest can quietly depend on fictional fills. Most dashboards visualize this problem; TISSUE operates it.

TISSUE is an autonomous fair-value and quote-policy desk built on TxLINE. It consumes the live scores and StablePrice odds streams, reconstructs match state, prices 1X2 and totals with a Poisson/Dixon-Coles model, detects event-to-market latency, and publishes two-sided quote recommendations only when edge survives feed-health, adverse-selection, model-divergence, inventory, exposure, and drawdown gates. When information quality drops, it halts instead of inventing certainty.

Solana is the evidence layer. Every live odds input is checked against the network’s daily root through validate_odds, while every decision-driving score update proves home/away goals and red cards through validate_stat. No source message enters the corpus or decision engine before validation, and recovery freshly revalidates persisted inputs. Each TISSUE decision is also hash-chained, so a judge can replay the exact corpus and reproduce the same ledger head. Live mode never claims a fill or PnL because TxLINE does not expose an orderbook; matching exists only in explicitly labelled replay research.

The system is three deployable services: a live-only daemon, a Next.js operating dashboard, and an isolated read-only analyst that narrates already-decided ledger evidence through three constrained tools. It fails closed without real credentials, persists crash-recoverable corpora, reconnects both SSE streams with dedupe and gap detection, and exposes health, readiness, verification, state, and event endpoints. Live decisions advance incrementally and append one ledger record instead of replaying the full fixture. The repository currently passes 120 tests, lint, all package typechecks, a zero-vulnerability production audit, deterministic replay, a production web build, and fresh local builds of all three pinned non-root images.

Next, we will capture current credentialed score/odds proof evidence, publish a multi-fixture holdout evaluation, and connect the quote output to a permissionless venue when TxLINE’s announced trading interface becomes available.

60-second judge path

  1. Open Overview and confirm the network, dual-stream health, proof counts, active fixture, and current desk state.
  2. Open Quotes to see Tissue fair value versus TxLINE StablePrice and every risk-approved output.
  3. Open Radar to inspect event-to-market reaction classifications and halt causes.
  4. Open Decisions to verify the hash chain and TxLINE/Solana evidence for each source message.
  5. Open Grade to inspect CLV, Brier calibration, latency, and ablation results from real fixtures.
  6. Ask the read-only analyst why the desk halted; inspect its exact ledger citations.

TxLINE feedback

The normalized StablePrice schema, stable message IDs, dual-stream design, and Merkle validation endpoints make deterministic autonomous tooling unusually practical. The main friction was discovery and documentation: activation returned plain text in the observed release, historical odds needed an in-play asOf, the free tier was totals-only, and the odds validation endpoint had to be found outside the initial quickstart. TISSUE records these exact findings without masking them with fallback data.
